# 安全政策 (Security Policy)

## 支援的版本 (Supported Versions)

目前我們為以下版本提供安全更新：

| 版本 (Version) | 支援狀態 (Supported) |
| -------------- | ------------------- |
| 1.x.x          | ✅ 支援              |

## 回報漏洞 (Reporting a Vulnerability)

如果您發現安全漏洞，請**不要**透過公開的 GitHub Issues 回報。

### 回報方式 (How to Report)

請透過以下方式私下回報安全漏洞：

1. **Email**: 發送郵件至 4b4g0077@stust.edu.tw
2. **主旨**: 請在郵件主旨標明 `[SECURITY]`

### 回報內容 (What to Include)

請在您的回報中包含：

- 漏洞的詳細描述
- 重現步驟
- 潛在影響評估
- 如果可能，建議的修復方案

### 回應時間 (Response Timeline)

- **確認收到**: 48 小時內
- **初步評估**: 7 天內
- **修復計畫**: 視漏洞嚴重程度而定

### 處理流程 (Process)

1. 收到回報後，我們會確認收到並開始評估
2. 我們會與您合作了解漏洞的性質和範圍
3. 開發並測試修復程式
4. 發布修復更新
5. 公開致謝（如果您希望的話）

## 安全最佳實踐 (Security Best Practices)

使用本專案時，建議：

- 定期更新到最新版本
- 不要在程式碼中硬編碼敏感資訊
- 使用環境變數管理 API 金鑰和密碼
- 在生產環境中使用適當的存取控制

## 致謝 (Acknowledgments)

我們感謝所有負責任地揭露安全問題的研究人員和用戶。
